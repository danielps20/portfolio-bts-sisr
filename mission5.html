🎯 Objectif

Mettre en place des ACL sur Router5 afin de contrôler les flux entre LAN, DMZ et Internet, avec filtrage et journalisation.

🧭 Topologie

  • LAN : 172.17.0.0/16
  • DMZ : 10.54.0.0/24
  • Internet : 160.250.0.0 → 193.253.0.0

Interfaces Router5

  • G0/0 → LAN
  • G0/1 → DMZ
  • G0/2 → Internet

🔐 ACL IN (LAN + DMZ)

ip access-list extended ACL_IN permit ip 172.17.0.0 0.0.255.255 any permit ip 10.54.0.0 0.0.0.255 any deny ip any any

Application sur G0/0 et G0/1 (in)

🌍 ACL OUT Internet

ip access-list extended ACL_OUT_INTERNET permit ip 172.17.0.0 0.0.255.255 any permit ip 10.54.0.0 0.0.0.255 any deny ip any any

Application sur G0/2 (out)

⚠️ Règles importantes

  • LAN → Internet autorisé
  • DMZ → Internet autorisé
  • Internet → LAN interdit
  • Internet → DMZ interdit

🧪 Tests

  • LAN → Internet (ping / HTTP)
  • DMZ → Internet (ping)
  • Internet → LAN (bloqué)
  • Internet → DMZ (bloqué)

📊 Vérification

show access-lists

✔ Les compteurs doivent augmenter lors des tests

🏁 Conclusion

Les ACL permettent de sécuriser le routage entre LAN, DMZ et Internet en appliquant une politique de filtrage stricte et traçable via logs.
