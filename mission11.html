Sauvegarde VM USB

☰
Profil Projets Veille Parcours

🎯 Objectif

Créer une méthode professionnelle pour sauvegarder une machine virtuelle VirtualBox sur une clé USB en utilisant tar + Zstandard (zstd), avec optimisation, automatisation et vérification d’intégrité.

🧠 Principe

  • Compression rapide avec zstd
  • Archivage avec tar
  • Optimisation multi-cœurs (-T0)
  • Redondance possible sur plusieurs supports
  • Vérification avec SHA-256

📦 Pré-requis

sudo apt install zstd pv -y
  • VM éteinte
  • Clé USB en exFAT ou NTFS
  • VirtualBox installé

⚙️ Commande principale

time tar -I "zstd -T0 -10" -cvf ma_vm.tar.zst "NomDeLaVM"

✔ Compression rapide et efficace

🚀 Décompression

tar -I zstd -xvf ma_vm.tar.zst

📊 Optimisation VM avant backup

VBoxManage modifymedium disk disque.vdi --compact

Réduit fortement la taille du disque virtuel avant compression.

🔁 Alias Bash (automatisation)

alias tarzst='time tar -I "zstd -T0 -10" -cvf' alias untarzst='time tar -I zstd -xvf'

Ajout dans ~/.bashrc puis : source ~/.bashrc

📁 Sauvegarde sur clé USB

cp ma_vm.tar.zst /media/$USER/NOM_CLE/

⚠ Vérifier que la clé supporte les fichiers >4Go

🔐 Vérification intégrité

sha256sum ma_vm.tar.zst > ma_vm.sha256 sha256sum -c ma_vm.sha256

✔ Permet de garantir que la VM n’est pas corrompue

⏱ Mesure des performances

time tar -I "zstd -T0 -10" -cvf ma_vm.tar.zst "VM"
  • real : temps total
  • user : CPU utilisé
  • sys : système

📈 Variante avancée (pipeline)

tar -cf - "VM" | zstd -T0 -10 -o ma_vm.tar.zst

💡 Bonnes pratiques

  • Éteindre la VM avant sauvegarde
  • Supprimer snapshots inutiles
  • Utiliser exFAT pour compatibilité
  • Faire une double sauvegarde (2 clés USB)

🏁 Conclusion

Cette méthode permet une sauvegarde rapide, compressée et fiable des machines virtuelles. L’association tar + zstd offre un excellent compromis entre performance et sécurité des données.
